1. PROBLEM STATEMENT
--------------------------------------------------------------------------------

Why are students and freshers not able to understand how and what to invest in early on?

India has over 50 million young adults entering the workforce every year, yet most lack
basic financial literacy. The gap between "having money to save" and "knowing where
to invest" creates a massive barrier to wealth building.

Key Challenges:
- Complex financial jargon (SIP, ELSS, NPS, market timing)
- Overwhelming number of investment options
- No personalized guidance based on individual financial profiles
- Fear of making wrong investment decisions
- Lack of hands-on experience with tracking finances

Finmate solves this by providing:
✓ Interactive risk profiling to understand personal appetite
✓ Curated portfolio templates designed for Indian beginners
✓ Transaction import and expense analysis
✓ AI-powered financial assistant (MulyaAI) for instant Q&A
✓ Progress tracking and visualizations

3. MULTI-AGENT ARCHITECTURE
--------------------------------------------------------------------------------

Finmate uses a Multi-Agent AI Workflow where specialized agents collaborate
to provide comprehensive financial guidance. Each agent has a specific role:

+---------------------+---------------------------------------------------+
| AGENT               | DESCRIPTION                                       |
+---------------------+---------------------------------------------------+
| MulyaAI (Chat Bot)  | Financial Q&A assistant with persistent memory.   |
|                    | Uses Groq LLM with custom system prompt.         |
|                    | Has access to user's profile, portfolio, and     |
|                    | transaction history for personalization.           |
|                    |                                       |
|                    | Features:                              |
|                    | - Persistent conversation context        |
|                    | - User profile integration           |
|                    | - Transaction history reference    |
|                    | - Financial expertise (tax, investing)|
|                    | - Concise response format           |
+---------------------+---------------------------------------------------+
| Portfolio Agent     | Generates personalized portfolios based    |
|                    | on user's risk profile and goals.     |
|                    | Uses Groq LLM to create custom     |
|                    | allocation strategies.             |
|                    |                                       |
|                    | Features:                              |
|                    | - Risk-based allocation           |
|                    | - Template selection             |
|                    | - Custom asset recommendations      |
|                    | - Growth trajectory planning     |
+---------------------+---------------------------------------------------+
| Risk Agent        | Analyzes user's risk appetite     |
|                    | through questionnaire responses.   |
|                    | Computes risk score (1-100).       |
|                    |                                       |
|                    | Features:                              |
|                    | - Questionnaire processing        |
|                    | - Score calculation             |
|                    | - Category classification       |
|                    | - LLM-generated advice          |
+---------------------+---------------------------------------------------+
| Asset Agent       | Fetches live market data and      |
|                    | provides asset recommendations.      |
|                    | Integrates with Yahoo Finance   |
|                    | for Indian stocks.               |
|                    |                                       |
|                    | Features:                              |
|                    | - Live price fetching            |
|                    | - Indian stock symbols            |
|                    | - Portfolio optimization       |
|                    | - Asset rebalancing suggestions  |
+---------------------+---------------------------------------------------+
| Transaction      | Processes uploaded financial     |
| Processing Agent | documents and extracts         |
|                    | transaction data.              |
|                    |                                       |
|                    | Features:                              |
|                    | - CSV/Excel parsing             |
|                    | - Auto-categorization           |
|                    | - Duplicate detection          |
|                    | - Balance calculation          |
+---------------------+---------------------------------------------------+
| Trim Agent       | Optimizes portfolio allocation  |
|                    | using 50/30/20 or custom     |
|                    | Needs/Wants/Savings framework.  |
|                    |                                       |
|                    | Features:                              |
|                    | - Budget framework             |
|                    | - Trim calculations           |
|                    | - Surplus identification      |
|                    | - Optimization suggestions   |
+---------------------+---------------------------------------------------+

AGENT COLLABORATION FLOW:

1. User registers → Risk Agent assesses risk profile
2. Profile complete → Portfolio Agent generates personalized portfolio
3. Portfolio saved → User can import transactions
4. Ongoing → MulyaAI answers questions referencing all data
5. Periodic → Asset Agent fetches live prices
6. Optimizations → Trim Agent suggests improvements

7. GETTING STARTED
--------------------------------------------------------------------------------

PREREQUISITES:
- Python 3.9+
- Node.js 18+
- PostgreSQL (local or cloud like Supabase)
- Groq API key

INSTALLATION:

Frontend:
  cd client
  npm install
  npm run dev     # Runs on http://localhost:5173

Backend:
  cd server
  pip install -r requirements.txt
  python setup.py  # First-time setup
  python run.py     # Runs on http://localhost:8080

ENVIRONMENT VARIABLES:
  DATABASE_URL=postgresql://user:password@host:5432/dbname
  GROQ_API_KEY=<your-groq-api-key>
  ALPHA_VANTAGE_API_KEY=<optional-for-stock-prices>
  PINECONE_API_KEY=<your-pinecone-api-key>
  PINECONE_ENVIRONMENT=us-east-1
  PINECONE_INDEX_NAME=finmate-transactions

7B. VECTOR DATABASE SETUP
================================================================================

The application uses Pinecone for vector-based transaction categorization.
This enables automatic categorization of bank transactions based on
description similarity.

1. CREATE PINECONE ACCOUNT
   - Sign up at https://www.pinecone.io/
   - Create a new index: finmate-transactions
   - Dimensions: 384 (for all-MiniLM-L6-v2 model)
   - Metric: cosine

2. SEED THE VECTOR DATABASE
   From the server directory:
   ```bash
   cd server
   python -c "from controllers.vector_db.upload_to_vector_db import seed_from_csv; seed_from_csv()"
   ```

3. EXPECTED CSV FORMAT
   If using your own data file (place in server/data/):
   ```csv
   description,category
   Salary,Income
   Shopping,Expenses
   Grocery,Food
   Rent,Housing
   ...
   ```

The vector DB automatically generates embeddings using the all-MiniLM-L6-v2
model and stores them in Pinecone for fast similarity search.
